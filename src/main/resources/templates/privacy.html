Here's the Privacy Policy for The Cloudy Book Club.

Data and The Cloudy Book Club

The Cloudy Book Club maintains and uses the following data

  • Book review data entered by users of the application (including book title, genre, rating and summary review).

    Data item Processing Processing purpose
    Book title Stored To be able to display the title (or name) of the book
    Genre Stored To be able to display the genre of the book (e.g. novel, autobiography etc)
    Rating Stored To be able rate the book as either great, good, ok, poor or terrible
    Summary review Stored To be able to see what the reviewer thought of the book. Users are asked to avoid "spoilers" (i.e. giving away key details of a novel's plot)
    Google Books Data Stored To be able "point" to a selected Google Books entry enabling the display of an appropriate book cover image, Google Books book review and (sometimes) a preview of the book on Google Books.

    Requesting data deletion The application developer is (in GDPR terms) both the "data controller" and "dataprocessor".

    If a user has created a book review then they have the ability to delete that book review at any time.

    If a user wants to request the amendment or deletion of any other book data in The Cloudy Book Club, please email support@cloudybookclub.com to contact the data controller and we'll agree what needs to be edited or entirely deleted.

  • Comments data on book reviews on The Cloudy Book Club - logged on users with the necessary permissions can comment on any other users book reviews.

    Data item Processing Processing purpose
    Comment Stored To be able to see a user's comments on a book review

    Requesting data deletion A user can always delete any comment that they have made on any book review. However, the comment will be replaced by "Comment deleted by __name__"

    If a user wants to request the amendment or complete deletion of any comment data in The Cloudy Book Club, please email contact the data controller and we'll agree what needs to be edited or entirely deleted.

  • Logon details when logging on to The Cloudy Book Club via Google or Facebook.

    Friends of the application developer are invited to log on to The Cloudy Book Club - after which the "data processor" may add permissions to their account such that book reviews can be added or comments made on existing book reviews. There is nothing to stop anyone else logging on to the application but unless and until the "data processor" adds permissions to that logon's account, then logging on gets a user no additional access.

    When you log on to The Cloudy Book Club, the application stores the following information (mostly provided by the Authorisation Server i.e. Google or Facebook):

    Data item Processing Processing purpose
    First name Stored To be able to display a friendly greeting when logging on (and to confirm your identity)
    Last name Stored To be able to display a friendly greeting when logging on (and to confirm your identity)
    Email address Stored To support communication to users if any change to The Cloudy Book Club will affect their usage of it
    The "provider key" on the authorisation server Stored To enable successful completion of logon processing
    Authorisation server used Stored To show the user whether they have logged on via Google or Facebook
    URL to user's avatar (if any) Stored To be able to display an avatar when logging on (and to confirm your identity) and to display along side book reviews and comments
    Last logon dates Stored So the user can see the last logon date/time from their account (to have confidence no-one else has used their account)
    User's access level Set by the administrator and stored So that a user who has been given elevated access rights can do more on The Cloudy Book Club

    Requesting data deletion If you no longer want The Cloudy Book Club to store the above data for you, please email support@cloudybookclub.com and we'll delete your data.

    If you want a copy of the above data that The Cloudy Book Club holds for you, please send a subject access request to support@cloudybookclub.com the data controller.

Cookies and The Cloudy Book Club

The Cloudy Book Club makes minimal use of cookies (and no use of local or session storage). If you choose to logon to the application a couple of cookies will be dropped by the application

  • A cookie named "CLOUDY-JWT" maintains a logon token for the user during their current logon session. This is a functional cookie - logged in access to the application won't work without it
  • A cookie named "XSRF-TOKEN" is used to protect the user against a type of hacking called cross site request forgery. This is a functional cookie - updates to the application data won't work without it
  • The current version of The Cloudy Book Club makes no use of any user tracking / tracking cookies - if you find any, please let me know!
